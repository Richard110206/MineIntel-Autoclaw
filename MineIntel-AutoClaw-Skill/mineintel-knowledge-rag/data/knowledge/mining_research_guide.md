# 矿业智能化科研知识指南

## 典型应用场景

矿井安全监测常见场景包括井下人员安全行为识别、安全帽和矿灯佩戴检测、皮带运输异物检测、矿车和人员区域入侵检测、烟雾火焰异常识别、仪表读数识别、井下巡检机器人视觉感知、瓦斯与温湿度多传感器异常预警。

适合大创或本科科研的题目应优先选择数据边界清楚、可演示、baseline 成熟的方向。例如“基于 YOLO 的煤矿皮带异物检测”“弱光粉尘条件下井下人员安全帽检测”“井下巡检机器人视觉感知与报警原型系统”。

## 井下环境技术难点

井下视觉和感知任务不同于通用室外或工厂场景。主要难点包括弱光照、局部强反光、粉尘和水雾遮挡、摄像头抖动、目标小且远、人员和设备互相遮挡、网络带宽不稳定、边缘设备算力受限、设备防爆合规要求、误报和漏报代价高。

做选题时不要只写“提升准确率”。更好的表述是围绕低照增强、小目标检测、轻量化部署、多模态融合、异常检测、误报抑制和边缘端实时性形成具体研究问题。

## 技术路线建议

计算机视觉方向可以从目标检测、语义分割、行为识别、异常检测和视觉增强切入。baseline 可选 YOLO 系列、MMDetection、Detectron2、OpenMMLab、PaddleDetection 等通用框架，再根据矿井数据做类别定义、标注格式转换、数据增强和模型轻量化。

传感器和安全预警方向可以从时间序列异常检测、瓦斯浓度预测、多源传感器融合、风险评分和报警策略优化切入。需要说明数据采集频率、缺失值处理、异常阈值和实时报警机制。

机器人方向可以从视觉感知、路径巡检、目标识别、仪表读数、远程告警和人机协同切入。大创阶段可先做仿真或小车原型，不要把完整井下无人系统作为本科项目目标。

## 评估指标

视觉检测类常用指标包括 precision、recall、mAP、F1、FPS、模型参数量、边缘设备延迟、低照/粉尘扰动下性能下降幅度。矿井安全场景尤其要关注漏报率和误报率，不能只看 mAP。

预警类任务常用指标包括 MAE、RMSE、AUC、召回率、提前预警时间、误报次数、报警延迟。报告中应解释指标与安全价值的关系。

## 数据与合规

如果没有真实矿井数据，可以采用公开数据集、公开视频片段、实验室模拟场景、合成数据或迁移学习方案。涉及学校官网、企业系统、知网、登录态平台时，不应自动登录或绕过验证码。优先使用公开网页、公开 API 和可复现实验数据。

## 报告写作要点

科研调研报告应包含问题背景、矿井场景特殊性、国内研究线索、国际前沿线索、开源 baseline、技术路线、数据方案、评估指标、风险与可行性、导师匹配和后续计划。搜索来源要保留，信息不足的论文线索只作为检索线索呈现。
