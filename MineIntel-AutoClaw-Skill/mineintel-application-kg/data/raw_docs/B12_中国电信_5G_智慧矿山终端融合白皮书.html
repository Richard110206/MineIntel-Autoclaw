01概要


能源是工业的粮食、国民经济的命脉。党的十八大以来，习近平总书记提出了“四个革命、一个合作” 能源安全新战略， 围绕推动能源消费革命、能源供给革命、能源技术革命、能源体制革命，全方位加强国际   合作，实现开放条件下能源安全，为新时代我国能源高质量发展指明了方向。国家能源局等8部委联合发布   《关于加快煤矿智能化发展的指导意见》， 到2035年实现工作面无人化或少人化， 我国将实现智能感知、智能决策、自动执行的无人化智慧煤矿生产体系。2021年国家能源局印发《智能化示范煤矿验收管理办法   （试行） 》的通知， 明确5G作为智能化煤矿的基础设施， 无线网络覆盖主要运输大巷、进风大巷、采煤工作面、掘进工作面等区域，开展5G+煤矿应用可获得相应加分项。


5G技术具备大带宽、低时延、广覆盖等特性，商用以来应用规模持续扩大，应用探索横向扩展，已从应 用试点向行业融合迸发。当前5G行业终端面临生产成本高、市场碎片化、产品适配差、行业融合难、跨行业标准缺乏等系列问题，严重影响5G行业应用规模化落地进程。


为促进5G行业规模发展，中国电信结合5G技术演进与细分行业特性提出“网络融合、终端融合、平台 融合”策略，终端融合是连接网络基础设施和行业应用平台的桥梁，通过分析行业典型业务及5G终端需 求，实现5G终端与行业需求的高度适配，以软硬解耦、设备云化为发展思路，实现5G行业终端的标准化和 规模化，降低终端成本，促进5G行业终端市场发展。


随着各行业对5G技术理解和应用的不断深入，越来越多的5G新特性得以应用，5G   LAN、5G高精度定 位、高精度授时、超级上行等技术逐步得到行业认可，亟待制定符合融合行业特性需求的重点行业5G终端 融合标准。


本文针对5G终端融合提出发展思路，对智慧矿山行业进行场景梳理及终端需求分析，融合中国电信现有 终端侧创新技术，制定5G+智慧矿山终端融合方案。


02   5G终端融合发展思路


“5G发展，终端先行”已成为行业共识。自5G商用以来，经过四年的摸索，5G在垂直行业的应用场景 已日渐清晰， 从试点探索转入快速推广阶段， 随着行业需求的释放及行业应用的铺开，5G行业终端需求和 市场空间都已极具规模。但在5G行业的实际落地和使用过程当中， 仍存在组网开局难、组网效率低、终端 改造难、安全要求高、可靠性要求高、管理运维难在内的6大痛点问题。


>> 01组网开局难：组网开局是行业终端接入5G网络面临的首要问题，终端设备的数量多、类型多、位置分 散。传统的行业终端存在安装工期长、部署调试复杂、施工要求多、可扩展性弱的问题。终端在完成物理上 的部署后往往还需要各种配置才能接入网络，操作繁琐容易出错，且对人员的专业要求较高，导致组网开局 困难。简易灵活的配置、极简自动化部署的网络、自动接入网络平台的能力，能够有效提升终端组网开局的 效率，是5G行业终端融合的重要诉求；


>> 02组网效率低：由于传统的OT技术手段不足，规划滞后等原因，传统行业中，大多是各子系统独立建网，   系统间信息不共享， 数据无法互通， 子系统间无法有效配合实现联动。所以新型5G专网建设，需要打破信 息孤岛，将数据汇聚在平台统一处理，实现数据共享，放大物联终端的行业价值，实现系统开放性，营造丰 富的行业应用环境；


>> 03终端改造难： 行业场景中存在大量的OT系统的终端和传感装置，这些终端需要进行数字化、IP化改造， 使之可以低成本、高效地接入网络，实现行业终端改造；


>> 04安全要求高：而物联终端分布广，终端CPU处理性能弱等特点，使得行业终端容易成为网络攻击的突破 口。对于大多数的行业应用场景，特别是在政府、教育、金融等领域，对于数据安全等级保护有明确要求。 行业终端低成本、高效地满足行业应用的场景需求，是5G终端融合需研究的一大课题；


>> 05可靠性要求高：行业场景对于终端的可靠要求更高，特别在工业制造领域，往往都是不间断持续的生 产，其中的终端需要更高的可靠性，才能保证业务连续；


>> 06管理运维难：行业终端数量多，线路布线复杂，   一旦出现故障，如果没有自动化故障检测和拓扑定位技 术，单靠人工排查，工作量大效率低，甚至出现故障后用户无感知，故障恢复时间长。所以对于物联终端， 做到管理运维智能化、拓扑呈现、GIS定位、故障预测，能够极大的降低运维管理方面的人力成本，提高行 业客户体验。


为应对5G行业终端的痛点问题，解决终端在功能、性能、成本和市场规模上的矛盾，中国电信提出了以 终端软硬解耦，能力开放， 终端云化为基础的5G终端融合创新发展策略， 促进终端与网络和平台的融合发 展，推动行业场景中端-网-云-用的建设，实现5G终端融合的落地和规模发展，并于2023年2月22日携手 终端产业链合作伙伴发布了“中国电信5G Inside合作计划”，以汇聚行业生态，构建5G产业生态核心竞争 力，形成更加丰富的5G行业终端产品体系为目标，以软硬解耦、能力嵌入、平台纳管、产品共创为原则，   提升5G与行业技术融合能力，驱动行业技术升级，加速5G终端与行业应用深度融合。中国电信5G   Inside 合作计划为产业合作伙伴提供五大权益，包括合作招募、产品导入、平台推广、方案优选、行业推介，积极 为合作伙伴赋能，激发5G生态合作新价值，与合作伙伴共同推进5G+ICT规模发展。


2.1.终端软硬解耦


2.1.1 软硬解耦技术架构


从20世纪80年代的计算机产业的软硬解耦推动IT行业的大发展与大繁荣，到21世纪10年代的以 SDN/NFV技术为基础重构传统电信网络，软硬解耦的技术贯穿着IT行业的发展，也是推动IT行业持续创 新、开放包容的核心技术。在传统的工业场景，软硬一体的终端仍然占据主流，工业协议不统一且数据开放 性不够，导致终端成为工业互联网规模化发展的一大瓶颈。通过终端软硬解耦，打破软硬一体化封闭的行业 终端体系，实现硬件通用化，服务软件化，提供万物智联的数字化基础。


终端软硬解耦技术是指将终端的硬件部分于软件功能分离，终端硬件上采用更为通用化的解决方案，主 要满足终端的通信、计算、存储、供电、防护等通用需求，通过定制化的软件赋能终端，满足各行业的定制 化功能需求。通过终端软硬解耦，打造5G行业终端通用硬件， 提升终端的行业融合属性。行业终端各种的 定制化功能以软件模块、SDK包等方式呈现，定制的能力模块可复用于多种的终端，达到一次开发、重复使 用的目的，可以降低5G行业终端的研发周期和生产成本。


中国电信将以开放的态度推动行业终端原子能力共享池的建设，打造包括了组网能力、云端协同能力、 数据处理能力、物联能力、安全管控能力等能力模块建设。终端原子能力随用随取，通过标准化接口快速赋 能终端，实现终端的快速开发， 一同推动5G终端融合的规模发展。



2.1.2   硬件基础  


行业终端的成本很大程度上取决于硬件的成本，行业场景众多，对于功能的需求也很分散，对于行业终 端往往需要定制化设计硬件架构，导致行业终端很难像芯片和模组一样通过规模化的生产降低成本， 间接导 致了行业终端的价格始终偏高。在终端软硬解耦后，终端硬件只需要满足基本的通信需求、接口要求、计算 能力要求、防护要求、功耗等，行业终端硬件设计时可以采用更为通用化的解决方案，适应更多更广泛的业 务需求，可以显著提升单一类型的行业终端的市场规模，进而降低研发和生产的成本。


例如在许多行业场景中都有广泛应用的5G网关、5G摄像头或AGV等终端，除个别的行业特性需求外， 不同场景对终端大部分硬件的需求基本一致，在设计通用硬件的架构时，在不显著影响终端成本的情况下， 可以将不同行业的个性化需求都考虑在内，形成一个满足多行业场景的终端通用硬件基础。


2.1.3   软件能力


在终端通用硬件基础之上，定制化的软件是定义终端业务功能的关键。中国电信已开展终端软硬解耦相 关的技术实践及原子能力的开发。例如：


>> 01 已开发协议库组件，支持超过150种的工业协议，可在OpenHarmony、FreeRTOS、RTThread、 lunix等系统上编译与适配，提供的对外API接口，目前已完成与传统网关设备和5G   CPE的结合，形成具备 协议转换功能的5G工业规约器和5G工业CPE。


>> 02 提出并开发终端与平台自适应功能。实现物联终端自动接入平台、软件自动更新，解决了物联网终端接 入困难，后期维护困难的问题。


>> 03开发探针拨测SDK包，可适配多种类型的终端硬件，快速赋能终端实现拨测功能，提升网络质量的监测 与故障定位能力。


2.2.设备云化   


2.2.1 设备云化架构


随着5G高速网络、高性能算力服务器、云计算/云原生等技术的发展，设备云化技术得以快速发展。设 备通过特定的5G网络连接到云平台，将原本终端设备的控制、计算、存储、AI、大数据、安全等能力通过 云平台处理，从而提升设备的业务服务能力。


设备云化最大的优势在于可以进一步实现终端硬件解耦，在设备侧可以减少对于控制芯片、CPU、AI芯 片、存储模块等硬件的需求。在应用上，云终端的应用开发者无需适配众多的终端操作系统，无需考虑因终 端型号与固件版本导致的应用升级难问题，可大幅减少应用安装、升级涉及的适配和测试的工作量。云终端 还具备多终端融合的特性，多终端业务体验的一致性及业务使用连续性将大大提升。



设备云化架构分为设备-边缘云-中心云三层，设备能力上移，降低终端硬件要求以及软件复杂度，对于 海量终端的场景，在终端成本优化和运维方面会有明显收益，通过边缘云可以支持设备的就近接入，实现数 据处理的低时延， 中心云负责云设备的资源调配。边缘云包括接入模块、功能模块和管理模块，接入模块负 责协议信令的交互以及设备的接入认证；功能模块包括了云化控制、云AI、云计算、云存储等功能，处理各 种的设备业务流；管理模块包括了设备的管理、运维的功能。


2.2.2   矿山场景下设备云化实践应用    


在工业互联网领域，机器换人是产业数字化的市场刚需，尤其在矿山场景中，存在大量的危险恶劣的生 产环境，人员下井需严格控制，通过机器代人的需求尤为强烈。


传统的工业一体化控制系统，通常采用单一总线通信，语言编程和编程系统由供应商定义，底层架构封 闭，与设备硬件紧锁定，功能单一，本地算力扩容困难，制约工业设备的数字化升级改造。可以通过5G+云 化设备方案对传统的矿山设备进行改造，提供无线数据的传输和工业设备的控制能力，支持远程控制、智能 监控等矿山业务场景需求。


（一）远程控制


安全生产作为矿山场景第一要素，当前亟待研发远程控制系统以取代人员实地控制的高风险生产模式。 通过5G+云化设备方案远程控制包括采煤机、掘井机、加渣机械臂、矿卡等典型设备，可广泛应用于工程机 械现场施工、机械臂实训及矿卡运输远程控制，实现“少人、无人、安全、高效”的矿山生产。



具体来说，5G+云化设备方案通过将设备主要功能进行模块化分解，将例如多轴协同、图像编解码、机 械臂移动、矿卡路线规划等部分关键功能从设备硬件中进行解耦，形成异构设备驱动库、云端标准组件库。 通过分布式部署的5G网络，利用集中管理平台进行工业控制应用管理、IDE应用开发，进而利用南北向不同 接口，实现矿山场景高精度操作控制、多设备快捷适配、多终端互控操作。


（二）智能监控


考虑矿业环境中设备运行状态、瓦斯浓度等对生产效率及人员安全的重要影响，5G+云化设备方案将终 端AI算法进行云化，进而对传统依赖人工或软硬件一体化系统的检测及告警方式进行升级， 实现以AI图像智 能识别技术为核心、以深度学习数学模型和报警系统为支撑、以自动化模糊控制技术为手段的智能监控功 能，在人员行为监管、重点区域监控、重点目标监管等多个矿山场景应用方面发挥作用。


更具体来说， 5G+云化设备方案将传统监控设备功能进行软硬解耦， 建立包含运动人脸/物体的检测识 别、图像抓取、目标跟踪和筛选等一系列功能的软件库。同时，通过在云端部署AI图像智能识别技术，可实 现人员、物品等具体矿业监控目标的高精度识别与检测。通过5G+云化设备方案，实现定制AI算法及工具、 多目标适配、机器视觉数据采集等功能需求，助力满足高效、高精度的智能监控矿业需求。



03   矿山典型业务场景及终端业务需求


3.1.矿山典型业务场景


3.1.1 露天矿                      


露天矿开采由剥岩、采矿和掘沟三个环节组成，各作业区域按功能不同主要可划分为采煤区、采掘平盘 区、回填区和破碎区四大部分。主要包括电铲远控、矿卡远程控制、辅助设备数据采集、无人机预警，视频 AI监控等业务需求。


露天矿对于终端类型的需求如下：


>>01 远程控制矿卡:采矿区，完成矿石运输工作。

>>02 电铲:操作员实时远程操控大型采掘设备，下发远程操控指令。

>>03 无人机:结合AI图像识别，对矿区进行日常巡检;爆破期间，爆破区域实现智能化警戒自动识别人、动物、矿车等后，及时告警。

>>04 监控摄像头:监控生产作业，图像实时回传。


3.1.2 井工煤矿                   


井工煤矿包含以下关键作业场景：


>> 01   综采面：采煤工作面， 一般2-4米高（取决于煤层的厚度，使用的采煤机不同）， 一个矿一般有2-3个工作面。


>> 02   掘进面：综采工作面形成初期由掘进机开辟通道， U型，一侧为运输顺槽， 一侧为回风顺槽，底边为综采面。


>> 03   硐室：机电设备存放，人员躲避，排水，纵深约100米以上。


>> 04   巷道：即主运巷道（运煤），辅运巷道（运人，物料），长度几公里至数十公里， 一般可以并排通行两辆胶轮车。


>> 05   全域：井上井下全域范围，人、车等的定位、实时通信和统一调度。     


井工煤矿对于终端类型的需求如下：


>> 01   “三机”：包括采煤机、刮板运输机，液压支架。

>> 02 巡检机器人：运煤皮带巷道、硐室，利用轨道巡检机器人，进行硐室、皮带巷等位置的巡检， 替代有人值守。



>> 03 有轨电机车：有轨电机车为蓄电池变频动力类轨道运输设备，在主要的运输巷道、辅助运输巷道、掘进巷道中对矿车进行牵引，完成原煤、材料、设备、人员等的运输任务。



>> 04 监控摄像头：用于主巷道等场景的高清视频监控。


>> 05 皮带AI巡检仪：运煤皮带巷道，利用摄像头或者巡检仪，辅以云端AI能力，进行皮带异物检测、 跑偏检测、探放水检测、室设备运行状态监测等，提升设备运行状态预报能力，从被动相应变    为主动预防。


3.1.3 非煤矿场景     


非煤矿场景分为地面开采和井下开采两种类型，具体场景包括：


>> 01   掘（爆破）：在矿体内装填炸药进行爆破，爆破后矿石形成矿石块；


>> 02   采（出矿）：通过矿石铲运机将矿石运输到溜井处进行汇聚；


>> 03   矿井内运输：通过电机车将矿石运输到卸载站，准备破碎。初碎：采用破碎机对矿石进行破碎， 便于皮带传输；


>> 04   运输至选厂：通过皮带传送将初碎后的矿石运输至选矿厂，进行选矿作业。


非煤矿场景包括以下终端类型：


>> 01   铲运车：将矿石运输到溜井处进行汇聚。



>> 02 其他终端需求：包括巡检机器人、监控摄像头、有轨电机车、皮带AI巡检仪可参考井工煤矿场景。

4.2   5G新特性需求


本节介绍的为矿山场景强相关的5G的新特性需求，其中对于终端的要求均为推荐性要求。


4.2.1   5G LAN   


5G   LAN能够向客户提供L3层基于IP VRF转发的VPN服务，以及L2层基于Ethernet局域网的LAN服 务。同时5G   LAN也支持用户移动性，支持细分子网，以及基于子网的管理能力。认证的用户可以随时随 地接入，不需要安装APP。5G LAN技术可以构建灵活互通和便捷管理的专网。


终端支持5G LAN需满足以下需求：


>> 01   支持网络通过UE Configuration Update procedure更新URSP（携带VN配置信息，如DNN， S-NSSAI， PDU session type）。


>> 02   支持二次鉴权。


>> 03   支持根据URSP规则，携带DNN、S-NSSAI信息，发起基于IP和Ethernet的5G LAN-type service。


4.2.2   5G高精度定位


5G定位技术在4G E-CID、OTDOA和UTDOA技术的基础上，结合5G网络大带宽和多波束的特性，进 一步支持了multi-RTT、UL-AoA和DL-AoD等技术。其中5G高精度定位主要包括DL-TDOA、 UL-TDOA、UL-AoA和multi-RTT技术，定位的水平维和垂直维定位精度可做到小于3m，端到端时延小 于1s。


终端需支持基于R16的高精度的下行定位参考信号测量与上报。


4.2.3   5G网络授时功能


终端通常通过GPS或北斗等技术完成授时，当终端不支持GPS、北斗，或卫星信号较弱时，可采用5G 网络授时功能，实现终端的授时。


支持5G网络授时需满足以下需求：


当支持获取R16定义的SIB9高精度时钟信号和1PPS高精度授时模式的至少一种能力时，其在信号强场 环境下授时精度的误差应小于等于1μs。


4.2.4 超级上行


在3GPP   Rel-16版本中，为进一步提升5G的上行利用率和容量，正式引入超级上行技术。在SUL、上 行CA和EN-DC三种上行增强技术的基础上开启Uplink Tx Switching功能，即为超级上行技术。

4.3.终端维护和管理要求   


业务保障、网络监控、问题定界等是矿山场景终端、网络和平台运维面临的核心问 题。除终端的本地维护管理外，更需要对终端传输的业务SLA情况进行检测和监控，降低 因终端因素导致对矿山业务生产的影响。


4.3.1 终端本地维护管理


终端支持近端进行终端的管理和维护，能力如下：


>> 01   支持本地web管理界面功能。

>> 02   支持DMZ、端口转发、桥模式。

>> 03   通过指示灯显示入网和脱网状态。

>> 04   支持进行入网DNN配置。

>> 05   终端管理界面支持显示IMSI、IMEI、入网IP、小区PCI、RSRP和SINR信息。

>> 06   终端管理界面支持终端超时搜网周期设置功能。

>> 07   终端具备日志功能，日志正确记录前面的配置操作信息、终端的入网状态、记录终端入网异常信息。

>> 08   终端管理界面支持恢复出厂设置功能。

>> 09   终端支持进行锁频锁网设置能力。

>> 10   终端支持通过OTA升级的能力。


4.3.2 终端支持平台管理能力      


为提升矿山场景下终端维护管理效率，解决异常场景故障定位定位诉求，终端支持通过中国电信5G定制 网运营平台维护，终端需支持以下能力要求：


>> 01   终端需支持自动接入中国电信5G定制网运营平台。


>> 02   终端支持在在管理平台进行纳管，能够成功接入终端管理平台并上线。


>> 03   终端支持上报终端基本信息至终端管理平台，包含如下信息：终端名称、终端型号、SN、IMEI、 IMSI、版本号信息、设备状态（正常、告警、离线）。


>> 04   终端支持上报终端入网信息至终端管理平台，包含如下信息：   RSRP/SINR/入驻小区PCI；接入频点、驻入小区信息、IP地址、邻区信号强度、MCS、S-TMSI（本项可选）等信息。


>> 05   终端支持上报业务指标至终端管理平台，包含如下信息： TCP/UDP上下行发包数、TCP/UDP 上下行速率、TCP/UDP连接往返时延等。


>> 06   终端支持通过终端管理平台下挂设备基本信息采集，包括：下挂设备状态（在线、离线），下挂   设备类型、下挂设备MAC地址等终端支持管理平台进行如下设置：远程修改参数（APN、网络制式选择、DMZ等），远程重启，远程调试等功能，并近端登录终端查看设置成功。 >> 07   终端支持维护链路异常断开后重连。


>> 08   终端支持通过终端管理平台远程升级。


>> 09   终端支持上报终端日志到终端管理平台，日志正确记录前面的配置操作信息、终端的入网状态、 记录终端入网异常信息。


4.3.3 终端支持企业业务管理能力               


为更好的管理矿山场景下终端传输的业务SLA的满足情况，终端支持进行业务SLA检测和监控，并上 报至企业业务管理平台，降低因终端因素导致对矿山业务生产的影响，终端需支持以下能力要求：


>>01终端支持通过企业业务管理平台下发TCP/UDP协议栈测速任务给终端，终端能够正常执行TCP协 议栈测试任务，并上报测试执行信息给平台包含：测试周期、丢包率、带宽信息。


>>02终端支持通过企业业务管理平台下发远程控制类业务模拟拨测任务给终端，终端能够正常执行拨测 任务，并上报远程控制类业务质量数据，指标包括：平均包时延、最大包时延、最小包时延、丢包 序号、丢包时刻、可靠包率、超时包率、重复包数、乱序包数。


>>03终端支持通过企业业务管理平台下发视频业务模拟拨测任务给终端，终端能够正常执行拨测任务，并上报视频类业务质量数据，指标包括：视频成功率、视频卡顿次数、视频卡顿时长、视频下载速率。


4.4.定位能力要求


5G接入网定义的标准定位技术分为蜂窝定位技术和非蜂窝定位技术两类。其中，蜂窝定位技术，包括如下定位方法：


>> 01   NR增强小区ID定位技术（NRE-CID）

>> 02   多环回传播时间定位技术（Multi-RTT）

>> 03   下行出发角定位技术（DL-AoD）

>> 04   下行到达时间差定位技术（DL-TDOA）

>> 05   上行到达时间差定位技术（UL-TDOA）

>> 06   上行达到角定位技术（UL-AoA）


非蜂窝定位技术，包括如下定位方法：


>> 01   网络辅助GNSS（A-GNSS）定位技术

>> 02   WLAN定位技术

>> 03   蓝牙定位技术

>> 04   地面信标系统定位技术

>> 05   基于传感器的定位技术：包括大气压力传感器、运动传感器

终端可根据实际场景需求支持合适的定位技术，详细要求可参考CCSA行标《5G移动通信网定位系统 终端设备技术要求（第一阶段）》。


4.5.机卡要求   


如下是对终端的要求：


>> 01   终端应支持使用中国电信5G卡。

>> 08   其它机卡要求参见《中国电信移动终端需求白皮书-机卡兼容功能分册》。


4.6.安全防护要求


矿山场景下终端安全防护的目标，主要包括实现环境适应性、硬件安全、操作系统 与固件安全、网络通信安全四个方面。


>> 01   环境适应性：矿山场景下对终端具有较强的本安、隔爆、防水、防尘等需求，终端应场景满足满足 环境适应性要求。


>> 02   硬件安全：确保设备芯片、外部接口等重要元器件免受攻击者的入侵或破坏。


>> 03   操作系统与固件安全：确保设备所使用的操作系统或固件能够安全地启动或运行，防止攻击者通过 本地或远程攻击的方式破坏操作系统或固件的正常运行。


>> 04   网络通信安全：确保设备与外界通信的安全性，防止攻击者对通信内容进行窃取、篡改或破坏。


4.7.终端接入安全


终端需具备接入的网络准入和访问控制能力，多重的接入控制能确保只有合法的终端才能接入5G网 络。对于通过5G    CPE或5G网关接入5G网络的终端，主要是通过在5G    CPE或网关上进行相关安全配 置，包括白名单认证、WiFi鉴权加密、启用防火墙等确保终端接入设备的安全。


对于直接接入5G网络的终端，终端的身份认证主要采用由核心网实现对终端基于5G   AKA或 EAP-AKA’的主认证。在核心网实现终端主认证的基础上，对于敏感行业用户可根据实际的安全风险， 可由企业侧基于AAA对终端开展二次认证，以加强企业用户对终端的自主控制。

5.2.2 矿山远程控制终端


矿山远程控制终端主要通过在设备上安装远程操控系统和监控与感知设备，对各类采矿设备进行远程 控制改造，利用5G技术的大带宽、低时延的特性，实时回传现场情况，并基于远程操作台和视频监控平 台，实现对设备的远程操作、控制。远程操控的设备包括电铲、掘进机、采煤机、钻机、胶轮车等。


对于远程操控终端的业务需求如下：


>> 01   应支持在阴天、潮湿、多粉尘的环境条件下进行作业，井下场景需满足本安、隔爆等要求。

>> 02   应支持移动类设备的定位能力，定位精度在亚米级。

>> 03   应支持对设备的位置运动轨迹监控，并反馈给控制室。

>> 04   应支持对远程露天开采设备的采集信息、平台控制信息、设备的控制指令的接收、 处理及转发的功能。

>> 05   应支持至少2-4路摄像头的视频实时回传。

5.2.3 矿用巡检机器人


矿用巡检机器人可大大减少人员下井，广泛用于各类矿山场景的巡检业务。挂载摄像仪、音视频采集 设备、多参数传感器、自动化机电设备、激光避障雷达等，可通过5G传输，可接入到矿山5G专网中，实 现高清图像的传输。


对于巡检机器人的业务需求如下：


>> 01   有可调节的伸缩臂，满足巡检需求。

>> 02   摄像仪，可见光200万像素，32倍光学变焦。

>> 03   激光避障雷达，发现障碍，立刻停止移动。

>> 04   全天候24小时运转，移动途中不会因为电池亏电而中断。

>> 05   具备搭载各种参数传感器的能力，用于激光甲烷、CO、CO2、O2、温湿度、烟雾监测。

>> 06   一键呼叫调度与双向对讲。

>> 07   AI智能异常识别联动报警。


5.2.4   5G无人机


5G无人机主要用于露天矿区巡检与地质勘测，可实现高效的数据采集和处理，安全开展矿区的 测量作业。


对于5G无人机的业务需求如下：

>> 01   支持厘米级精度的仿真和定位能力。

>> 02   支持飞行路线自动规划和执行。

>> 03   支持人员识别、地形分析、环境行为监测。

>> 04   支持自动回程充电。


5.2.5 矿山智能监控摄像头         


矿山智能监控摄像头主要用于矿山各类场景下的静态监控场景。


对于矿山智能监控摄像头的业务需求如下：


>> 01   应支持至少1080p的分辨率，推荐支持2K以上的分辨率。

>> 02   支持录像功能、存储功能、文件查看功能、抓图功能、日志等。

>> 03   摄像头应支持在上电后自动发起网络连接；由于外部原因引起网络中断时，摄像头应能够周期性 发起连接尝试。

>> 04   支持夜间监控，夜间监控距离大于30米。

>> 05   应满足本安、隔爆、防尘、防水等要求。

>> 06   支持与远程固件升级平台对接，且在升级失败情况下具备恢复机制；支持通过HTTP、MQTT、 TR069等协议方式与相应的业务设备平台进行连接。

>> 07   推荐支持AI神经网络算法，支持运动人脸和物体的检测、跟踪、抓拍和筛选。

>> 08   推荐支持环状组网及网络冗余，支持多基站之间的直连和无线组网。

>> 09   推荐支持单独划分VLAN或组建监控专网。

>> 10    其他性能及接口要求可参考《中国电信移动终端需求白皮书-5G行业终端分册》5G摄像头部分。


5.2.6 矿山智能信息矿灯


智能信息矿灯可实现井下人员位置、环境参数、移动音视频等多源异构数据的实时采集与在线传输， 弥补井下监控盲区，提高作业人员应急保障能力，形成实时通信、事件溯源、应急调度、检查执法等不同 场景的一体化解决方案，实现矿井“实时预警、智能动态感知、安全管控”的目标。


对于智能信息矿灯的业务需求如下：


>> 01   具备照明功能，具有远光、近光切换功能。

>> 02   具备定位功能，静态精度≤0.3米。

>> 03   具备拍照摄像功能，200万以上像素，具有灯头和机体双摄像头，支持低照度摄像。

>> 04   具备数据无线传输功能，支持5G NR、WiFi IEEE 802.11 a/b/g/n/ac、蓝牙BT5.1 BLE等通信 协议。

>> 05   续航能力方面，电池总容量应不小于10Ah，整灯工作时间应不小于11h。

>> 06   具备查询功能，支持历史曲线、历史数据查询。

>> 07   具备移动视频监控功能，支持井下的智能矿灯设备与井上的远程指挥人员通信。

>> 08   具备应急广播功能、通话调度功能。

>> 09   具备一键拍照、 一键录音、 一键摄像等功能。

>> 10   防护等级： IP65。


5.2.7   智能健康监测设备


智能健康监测设备可实现矿工身体状况参数感知，通过将数据实时传输到矿用智能手机、智能信息矿 灯等终端，进而采用矿用5G/4G/WiFi等网络将数据实时传输到地面服务器端，实现人员安全状况识别，   实现对矿工异常状态的实施监测及预警功能，具有安全监测、体感监测、疲劳状态监测等功能，提高煤矿 管理者对矿工人员健康状态的掌控能力。


对于智能健康监测设备的业务需求如下：


>> 01   能够全天候连续测量井下人员的生命体征和健康状态数据，如心率数据、实时动态血压、体温、 血氧、运动距离、卡路里消耗情况等。

>> 02   可监测睡眠质量。

>> 03   可通过蓝牙交互技术将数据传到井下智能终端的客户端。

>> 04   具备蓝牙语音通话功能。

>> 05   可接收平台下发文本消息。

>> 06   具备消息提醒及显示、短信、来电、查看天气等功能。

>> 07   待机时间应不小于48h。

>> 08   通话时间应能够不小于2h。


06   矿山场景5G终端典型案例求


6.1 5G+无人驾驶系统


5G+无人驾驶系统包含矿车无人驾驶、无人运输仿真、远程驾驶、挖机协同作业管理、V2X智能路侧协 同系统、云端智能调度与管理六大子系统。六大子系统已实现软件硬件化、服务产品化、产品系列化和系统 平台化，可通过平行仿真系统虚实交互提升安全效率。子系统的搭配组合支持各类场景需求，并全部适配国 内外各型矿车，远程驾驶系统可独立运行保障安全。



6.2 矿用5G无线转发器


矿用本安型5G DTU，可实现5G信号转以太网信号或以太网转WiFi 6信号。兼容各种工业通讯协议，提 供丰富接口，满足各种工业应用场景。提供IOT平台，实现模组的升级、维护、告警等功能。客户无需软硬 件设计开发，即插即用，大幅降低了客户二次开发的成本和产品上市时间。


矿用隔爆兼本安型5G   CPE，支持4G/5G全网通，为用户提供无线长距离大数据传输功能。采用高性能 的工业级32位通信处理器和工业级无线模块，以嵌入式实时操作系统为软件支撑平台，可同时连接以太网设 备和WiFi设备，实现数据透明传输和路由功能。


6.3 5G巡检机器人


矿用本安式5G巡检机器人具有体积小巧、爬坡续航能力强、多项监测、WiFi/5g传输以及井下充电等特 点。机器人井下巡检时会对自身的电量进行监测， 当电量低于设定值时，会自动运行到充电仓处进行充电， 同时电量充足的机器人驶出充电仓，从而实现井下不间断巡检。


巡检机器人主要有四大应用场景：应用于皮带巡检时，可以对皮带跑偏和异物进行识别，保障设备正常 运行；应用于变电所时，机器人可以在禁止人员接近的区域正常巡检，检查设备运行状态； 同时可以对人员 进出进行报警，对开关设备进行识别；应用在泵房中，机器人可以检测泵站仪表信息， 同时对漏液等事故进 行智能识别；应用于工作面巡检时，可以监测采煤时的瓦斯浓度、工况视频， 以及采煤机的运行温度，维持 工作面各设备的正常运行。



6.4 矿用5G摄像仪


矿用5G摄像仪，支持5G网络传输，采用本安设计，适用于矿山场景。支持大算力、应用场景分析，高 分辨率、电动镜头、夜间摄影等功能。可用于智能视频分析、煤矿石检测、违规预警、异常告警等场景。



6.5 5G工业融合网关


中国电信自研5G工业融合网关系列， 支持各类工业设备网口、串口， 5G上网。支持超过150种工业协 议， 内置安全芯片，具备协议转换能力，支持本地存储，规则引擎，定时任务等功能，且可以深度定制丰富 的第三方应用和插件，功能可扩展性强。串口全隔离、所有端口防雷防浪涌、电源正反接保护，宽幅电源输 入，可定制化设计防护能力，满足工业场景中防水、防爆、防尘的要求。适用于工业数采、数控等场景，可 用于传统终端设备的无线化、数字化改造，提供便捷的接入5G网络及管理平台的能力。



6.6 矿用5G智能信息矿灯


矿用5G本安型智能信息矿灯，采用一体化设计，集成照明、通话、人员定位、拍照摄录、无线通信等模 块，实现井下移动监测、应急调度、智能巡检、事件溯源等场景应用。



6.7 矿用本安型智能健康监测终端及系统


矿用智能健康监测终端采用本安化设计，包括矿用本安型手环、矿用本安型手表，能够实时监测矿工体 征参数，将数据上传至安卓终端进而传输到地面服务器端，智能健康监测平台对矿工体征参数数据可视化， 供管理调度人员监测并辅助分析风险，紧急情况下可采取调度和紧急施救等措施， 为矿工的生命健康保驾护 航，保障生产安全。



6.8 5G组网型政企网关


中国电信自研5G组网型政企网关采用高性能工业级硬件平台，保障系统正常运行。外设接口丰富，支持 RS232、RS485、USB3.0、DI、DO、RJ45口等多种外设接口，且具备高防护等级电气性能。兼容 5G/4G网路，带有5个千兆以太网口，支持802.11a/b/g/n/ac/ax等协议，以及WiFi 2.4/5.8G双频段，支持 一键MESH组网功能， WPS功能，可轻松组建高速、稳定的有线、无线传输网络，并可利用公用5G/4G网 络为用户提供无线长距离数据传输功能。支持配置GRE、L2TP、IPSEC、EOIP、VXLAN等VPN功能， 支持从平台对终端进行管理和维护，查看终端运行转态、故障定位、远程升级、参数配置。可广泛应用于备 份专线、企业远程安全办公、多分支跨域快速组网、远程内网设备调试、远程控制、工业互联等场景。



07 产业展望


2022年9月，工信部印发《5G全连接工厂建设指南》,指出在“十四五”期间，面向原材料、装备、电 子等制造业各行业以及采矿、港口、电力等重点行业领域， 建成1000个分类分级、特色鲜明的工厂，打造 100个标杆工厂，推动5G融合应用纵深发展。实现生产单元广泛连接、信息（IT）运营（OT）深度融合、 数据要素充分利用、创新应用高效赋能的新局面。


为实现5G技术深度融合矿山行业，构建多系统集成的智能化矿山体系，中国电信希望通过与产业链通力 合作，共同探索5G+智慧矿山网络融合演进路径，推进5G由“辅助生产”向“融入生产”演进，进而达成 “改变生产”的美好愿景。


一、推动跨行业标准体系建设


构建5G智慧矿山融合专网体系，需要探索建立统一的行业融合标准和产业结构，形成统一的技术演进路 线。中国电信将联合产业链合作伙伴，依托3GPP、CCSA、工业互联网产业联盟等组织，面向露天矿、井 工矿等场景，开展技术攻关、标准制定、方案制定，打造跨行业标准体系。


二、深度促进产业技术融合


繁荣5G智慧矿山生态，需尽快推进5G芯片、模组、网关与矿山远控设备及矿山生产设备的研发与融合 应用，尽快加速相关厂商的产业技术融合，降低相关设备的成本。中国电信将通过5G产业创新联盟，聚合 产业伙伴，积极推动产业各方交流与合作。

三、试点创新行业应用


加快5G智慧矿山融合专网应用，积极促进创新成果在矿山行业推广应用。中国电信将借助OpenLab实验室，提供开放共享的测试环境，打造从网络验证到应用的端到端应用解决方案，在智慧矿山行业内试点创 新应用。
