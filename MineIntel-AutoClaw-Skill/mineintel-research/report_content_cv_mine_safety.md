# 计算机视觉在矿井安全监测中的大创选题科研调研报告

## 1. 研究主题概述

本报告面向软件工程专业学生，围绕“计算机视觉在矿井安全监测中的大创选题”进行科研调研。建议将选题收敛为：

**面向矿井低照度复杂场景的多目标安全隐患识别与边缘部署系统研究**

该选题以矿井视频图像为主要输入，重点识别人员安全装备佩戴、人员/车辆/障碍物、巷道裂缝或片帮、输送带火灾/异物等可视觉化风险。软件工程专业的优势在于可以把算法、数据管理、边缘推理、Web 可视化和报警闭环做成完整原型，而不是只做单个模型精度对比。

## 2. 研判结论摘要

1. **选题可行性高**：矿井安全监测已经具备大量固定摄像头、巡检机器人、视频监控和智能化矿山建设需求，计算机视觉可以作为现有传感器监测体系的补充。
2. **大创切入点应小而闭环**：不建议一开始覆盖“全矿井安全监测”。更适合从“井下 PPE 佩戴合规检测 + 低照度图像增强 + 边缘端实时预警看板”切入。
3. **核心难点不是模型名称，而是矿井场景适配**：低照度、粉尘、水雾、遮挡、小目标、视角固定但场景变化大，都会影响检测鲁棒性。
4. **推荐技术路线**：YOLO 系列目标检测作为 baseline，叠加低照度增强、注意力/轻量化模块、异常样本数据增强、边缘端部署和告警系统。
5. **申报材料重点**：强调矿井安全价值、软件系统闭环、可复现实验设计、数据合规与现场风险边界。

## 3. 应用场景概述

### 3.1 人员安全装备与不安全行为监测

矿井人员 PPE 佩戴、人员进入危险区域、人员倒地/滞留、人员与车辆距离过近，属于最适合视觉感知切入的安全监测方向。已有公开论文和项目多围绕安全帽、反光衣、自救器、人员目标检测展开。

### 3.2 巷道巡检与固定摄像头异常识别

在主运输巷道、皮带沿线、掘进工作面、转载点等场景，摄像头或巡检机器人可以识别障碍物、裂缝、设备异常、积水、火焰或烟雾。公开研究显示，井下低照明和环境杂乱会显著降低通用目标检测算法效果，因此需要面向矿井场景优化。

### 3.3 输送带与机电设备安全监测

煤矿胶带机异物、跑偏、堆煤、火灾和大块煤检测是矿井视觉监测的典型工程方向。对大创而言，可以选择“皮带异物/火焰检测”作为第二阶段拓展任务。

### 3.4 与瓦斯、顶板、矿压等传感器联动

瓦斯和顶板灾害本身不完全依赖视觉，但视觉可用于人员行为、现场状态和报警复核。例如瓦斯超限后联动摄像头确认人员撤离情况，顶板风险区域联动识别人员进入、设备停留和裂缝变化。

## 4. 矿井特殊技术难点

1. **低照度与光照不均**：井下照明弱、局部强反光、灯带闪烁，导致目标边界不稳定。
2. **粉尘、水雾和镜头污染**：图像质量下降，安全帽、人员、自救器等目标可能被遮挡。
3. **小目标与密集遮挡**：安全帽、自救器、裂缝、火焰早期区域通常尺寸小，容易漏检。
4. **数据采集难**：真实矿井数据涉及安全生产、隐私、企业保密和标注成本，公开数据集不足。
5. **边缘部署约束**：井下网络、算力、供电、防爆设备要求使模型必须轻量化。
6. **安全关键系统的误报/漏报代价高**：不能只展示 mAP，还要关注召回率、误报解释、报警延迟和人工复核机制。

## 7. GitHub baseline 推荐

**推荐 baseline：ultralytics/ultralytics**

- 地址：https://github.com/ultralytics/ultralytics
- 选择理由：生态成熟、训练/验证/推理/导出流程完整，支持 YOLO 系列检测、分割和姿态任务，适合软件工程学生快速构建“数据集-训练-部署-接口-看板”闭环。
- 迁移方式：
  1. 先用公开 PPE/安全帽数据集复现实验；
  2. 再整理矿井低照度增强数据或少量自采样例；
  3. 类别设置为 `person`、`helmet`、`no_helmet`、`vest`、`vehicle`、`obstacle`、`fire/smoke` 等；
  4. 导出 ONNX 或 TensorRT，用 Flask/FastAPI + Web 前端做实时预警演示。

可补充关注但不作为主 baseline 的仓库：

- `xgli411/MH-YOLO`：与煤矿安全帽检测论文关联，需核验仓库可用性。
- `970334745/AMI-YOLO`：与矿工 PPE 检测预印本关联，需核验代码完整性。
- `open-mmlab/mmdetection`：适合做更严谨的检测算法对比，但学习成本高于 Ultralytics。

## 9. 技术路线

### 9.1 最小可行版本

1. 收集公开 PPE/安全帽/工业安全数据集，补充少量矿井低照度公开图片或模拟数据。
2. 使用 Ultralytics YOLO 训练人员、安全帽、反光衣、车辆、障碍物等目标检测模型。
3. 引入低照度增强模块，例如 Retinex、Zero-DCE、CycleGAN 风格增强或简单亮度/噪声增强。
4. 构建视频流推理服务：摄像头/视频文件输入，模型输出检测框、置信度和报警事件。
5. Web 看板显示实时画面、事件列表、报警规则、统计图和历史截图。

### 9.2 进阶研究版本

1. **小目标增强**：增加 P2 检测头、注意力机制、FPN/PAN 改进或高分辨率裁剪。
2. **矿井域适配**：低照度、粉尘、运动模糊、遮挡的数据增强；跨数据集测试泛化能力。
3. **边缘部署**：模型剪枝、量化、ONNX/TensorRT 导出，比较 FPS、延迟和显存占用。
4. **报警策略**：将单帧检测改成时序事件，例如连续 N 帧未戴安全帽才报警，降低误报。
5. **安全评估**：重点报告召回率、漏检案例、误报原因和人工复核机制。

### 9.3 软件工程产出

1. 数据集管理脚本：划分训练集/验证集/测试集，标注格式转换。
2. 模型训练与实验追踪：保存参数、权重、日志和指标。
3. 推理服务：REST API 或 WebSocket 视频流。
4. 前端看板：实时检测画面、报警记录、风险统计。
5. 部署文档：本地 GPU、普通笔记本和边缘盒子的运行说明。

## 10. 大创项目建议书骨架

### 项目名称

面向矿井低照度复杂场景的多目标安全隐患识别与边缘预警系统

### 研究目标

1. 构建矿井安全视觉监测原型系统；
2. 实现人员 PPE、障碍物、车辆、火焰/烟雾等目标识别；
3. 研究低照度增强和轻量化检测对矿井场景的提升；
4. 完成 Web 可视化报警平台和边缘推理部署实验。

### 创新点

1. 将矿井低照度增强、目标检测和边缘部署合成闭环系统；
2. 以安全事件为中心设计时序报警规则，而不是只做单帧检测；
3. 面向软件工程专业，强调工程可复现、可部署和可演示。

### 评价指标

- 检测指标：mAP、Precision、Recall、F1；
- 安全指标：漏报率、误报率、报警延迟；
- 工程指标：FPS、模型大小、显存占用、部署耗时；
- 系统指标：事件记录完整性、前端展示稳定性、接口响应时间。

## 11. 科研经验参考

知乎/小红书等平台只作为大创推进经验参考，不作为论文或技术事实依据。可参考的执行建议：

1. 选题不要过大，题目里最好能明确“对象 + 场景 + 方法 + 系统”。
2. 先复现 baseline，再做小改进；大创答辩更看重完整闭环和过程材料。
3. 数据来源要合规，不能使用涉密井下视频；可用公开数据、模拟场景和企业授权样例。
4. 每周固定沉淀实验记录：数据版本、训练配置、指标、失败案例。
5. 答辩材料要准备“为什么不是普通安全帽检测，而是矿井安全监测”的场景差异说明。

## 13. 最终建议

建议以“矿井低照度 PPE 与隐患目标检测系统”为大创主线：第一阶段复现 YOLO baseline，第二阶段加入低照度增强和小目标优化，第三阶段完成边缘推理与 Web 报警看板。该路线既贴合软件工程专业，又能体现计算机视觉在矿井安全监测中的场景价值和工程落地能力。
