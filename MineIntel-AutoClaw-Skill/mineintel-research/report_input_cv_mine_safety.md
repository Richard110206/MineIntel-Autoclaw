# 计算机视觉在矿井安全监测中的大创选题调研

## 1. 研究主题概述

本次调研面向软件工程专业学生，技术领域为计算机视觉，矿业场景为矿井安全监测。建议把大创选题收束为：

**低照度矿井场景下人员安全装备与危险行为视觉监测原型系统**

该题目比泛泛的“计算机视觉在矿井安全监测中的应用”更适合大创执行，因为它同时具备清晰对象、可获取数据、可复现 baseline、可展示软件系统和可量化指标。项目可以从安全帽、自救器、人员越界、钻场作业交互或皮带异物检测中的一个对象切入，先完成目标检测 baseline 复现，再围绕矿井低照度、粉尘、水雾、遮挡、小目标和边缘端部署做适配优化。

政策和产业背景上，国家能源局《煤矿智能化标准体系建设指南》把平台与软件、算法智能平台、智能视频系统等列为煤矿智能化标准体系的重要组成部分，说明“视觉算法 + 软件平台 + 安全闭环”不是孤立算法题，而是煤矿智能化建设中的工程需求。公开产业方案中，掘进面作业智能监测也已将机器视觉目标识别、动作检测、传感与设备状态联动作为安全生产支撑方向。

## 2. 研判结论摘要

1. **选题可行**：计算机视觉与矿井安全监测存在明确交叉点，已有中文综述、国际论文、数据集和开源检测框架支撑。
2. **软件工程匹配度高**：项目不应只训练模型，而应形成“数据处理 - 模型训练 - 推理服务 - Web 可视化 - 报警规则”的软件闭环。
3. **创新点应聚焦场景适配**：不要把创新写成“使用 YOLO”，而要写成“面向低照度、粉尘遮挡、小目标和边缘端部署的矿井视觉检测适配”。
4. **大创周期内建议单点突破**：优先选择安全帽/自救器佩戴检测、井下人员检测、钻场作业交互检测或皮带异物检测之一，不要同时覆盖所有矿井灾害。
5. **实验风险主要在数据**：公开矿井数据有限，必须提前准备公开数据、仿真增强、公开视频截帧、校内合作采集或半合成数据方案。
6. **导师联系建议**：优先联系机器视觉、智能矿山、感知矿山或工业 AI 方向老师，邮件中突出你能承担软件系统、模型复现和实验记录工作。

## 3. 应用场景概览

### 3.1 人员安全装备与危险行为监测

面向安全帽、自救器、矿灯、反光服、越界进入危险区域、摔倒、违规操作等对象。该方向最适合大创，因为目标定义清晰，能用目标检测、姿态估计、跟踪和规则引擎做演示。国际公开 DsDPM 66 数据集已经包含 coal miner、mining helmet、compressed oxygen self-rescuer 等类别，可作为开题阶段的重要数据线索。

### 3.2 矿井视频图像增强与移动目标监测

矿井视频往往存在低照度、局部过曝、粉尘、水雾和灰黑背景问题，直接影响后续人员检测与设备识别。图像增强、低照度恢复、去雾去噪和检测任务耦合优化，是计算机视觉进入矿井安全监测的基础环节。

### 3.3 皮带运输与设备异常识别

面向煤流运输、皮带跑偏、异物、堆煤、人员靠近危险设备、设备异常状态等场景。该方向更接近工业检测和边缘推理，可以作为进阶方案，但数据采集和标注难度高于人员安全装备检测。

### 3.4 掘进面、钻场与巡检机器人感知

面向钻杆、钻机、人员与钻杆交互、掘进工序、巡检机器人导航和障碍物识别。该方向技术价值高，能结合目标检测、动作识别、三维姿态估计和多模态感知，但大创阶段建议只做一个可验证子任务。

## 4. 矿井特殊技术难点

1. **低照度与局部过曝并存**：井下人工照明不足，光源附近又容易过曝，导致目标纹理丢失。
2. **粉尘、水雾和噪声干扰**：煤尘、水雾、镜头污染会降低图像对比度，影响检测稳定性。
3. **小目标和遮挡**：安全帽、自救器、矿灯等目标面积小，人员与设备遮挡频繁。
4. **数据标注成本高**：真实矿井数据有安全和隐私限制，公开数据集数量有限。
5. **边缘端算力受限**：井下实时监测要求低延迟和高稳定性，模型需要轻量化。
6. **误报漏报成本高**：矿井安全系统不能只追求 mAP，还需要解释报警依据、降低漏报风险。
7. **系统闭环复杂**：算法结果需要进入前端看板、报警规则、日志追踪和人工复核流程。

## 7. GitHub baseline 推荐

**最终推荐 baseline：`ultralytics/ultralytics`**

- 仓库链接：https://github.com/ultralytics/ultralytics
- 推荐理由：YOLO 系列工具链成熟，适合快速完成目标检测训练、推理、跟踪、导出和 Web 演示接入；DsDPM 66 论文也把 Ultralytics YOLOv8 作为数据集验证模型之一，说明它适合作为矿井对象检测的第一基线。
- 适配任务：安全帽/自救器检测、人员检测、钻机/钻杆检测、皮带异物检测、危险区域越界检测。
- 软件工程价值：可以围绕数据集管理、训练脚本、推理 API、前端可视化、报警日志和模型版本管理形成完整系统。
- 风险提示：Ultralytics 使用 AGPL-3.0 与企业授权双轨许可，课堂、大创和开源演示通常可作为研究 baseline，但商业化或闭源部署前必须核验许可边界。

不建议把多个 GitHub 仓库同时写成主 baseline。若后续要拓展红外/热成像或 RGB-T 检测，可另行调研 YOLOv11-RGBT 等多光谱项目，但本次报告正式 baseline 保留一个，避免申报材料显得发散。

## 9. 技术路线

### 9.1 题目收束

推荐题目：

**低照度矿井场景下安全帽与自救器佩戴检测及可视化预警原型**

备选题目：

1. **面向矿井钻场作业的人员-设备交互视觉识别方法研究**
2. **粉尘遮挡环境下煤矿井下人员小目标检测与跟踪方法研究**
3. **基于 YOLO baseline 的矿井视频安全隐患识别 Web 原型系统**

### 9.2 数据路线

1. 公开数据优先：DsDPM 66 中 coal miner、mining helmet、compressed oxygen self-rescuer 等类别。
2. 补充数据：公开矿井视频帧、工业安全帽数据、低照度数据集、校内模拟采集。
3. 数据增强：低照度、模糊、粉尘噪声、遮挡、裁剪、小目标缩放、颜色偏移。
4. 标注格式：YOLO 格式为主，必要时转换 COCO 格式，便于和 DETR/Swin 等方法比较。

### 9.3 模型路线

1. 复现 `ultralytics/ultralytics` YOLO baseline，得到初始 mAP、Precision、Recall、FPS。
2. 进行矿井适配：低照度增强前处理、Mosaic/Copy-Paste 小目标增强、类别不平衡重采样、轻量化模型选择。
3. 引入跟踪：对人员连续越界、未佩戴装备持续时间进行规则判断，减少单帧误报。
4. 做对比实验：原始图像 vs 增强图像，通用训练 vs 矿井增强训练，大模型 vs 轻量模型。

### 9.4 软件系统路线

1. 后端：Python 推理服务，提供视频上传、帧推理、报警日志接口。
2. 前端：Web 看板展示视频帧、检测框、类别、置信度、报警列表和统计曲线。
3. 数据管理：保存样本、标签、训练配置、模型版本和评估结果。
4. 报警规则：例如连续 N 帧未检测到安全帽、自救器缺失、人员进入危险区域。
5. 交付演示：短视频演示、模型指标表、典型失败案例和改进建议。

### 9.5 评估指标

- 检测指标：mAP@0.5、mAP@0.5:0.95、Precision、Recall、F1。
- 工程指标：FPS、单帧延迟、模型大小、显存/内存占用。
- 安全指标：漏报率、误报率、连续报警稳定性、低照度场景表现。
- 软件指标：接口稳定性、前端展示完整性、日志可追溯性、配置可复现性。

## 10. 科研经验参考

知乎、小红书等平台只建议作为项目推进经验参考，不作为论文事实依据。可参考的执行建议如下：

1. 选题要小而闭环：不要写“矿井智能安全监测大系统”，应写一个能训练、能演示、能解释的对象。
2. 申报书要突出“为什么矿井场景不一样”：低照度、粉尘、水雾、小目标、遮挡、边缘部署。
3. 组队建议：算法训练 1 人，前端/后端 1 人，数据标注与文献 1 人，答辩材料 1 人。
4. 每周沉淀产物：文献卡片、数据样例、训练日志、错误案例、系统截图。

## 12. 最终建议

建议将项目定位为“软件工程驱动的矿井安全视觉监测原型”，核心贡献不是发明一个全新检测模型，而是把公开数据、YOLO baseline、矿井场景增强、推理服务和 Web 报警看板串成可复现实验闭环。大创阶段最优路线是：

1. 第一阶段：文献综述和数据集整理，完成 20 篇文献卡片和 1 个公开数据 baseline。
2. 第二阶段：复现 YOLO 检测，输出训练日志、指标表和失败案例。
3. 第三阶段：针对低照度/粉尘/小目标做一个可解释改进。
4. 第四阶段：完成 Web 原型和报警规则，准备答辩演示视频。
5. 第五阶段：联系导师，争取数据、场景和项目申报建议。

核心一句话：**从“我会用计算机视觉”升级为“我能把矿井安全场景中的一个视觉问题做成可验证、可演示、可复现的软件工程闭环”。**
