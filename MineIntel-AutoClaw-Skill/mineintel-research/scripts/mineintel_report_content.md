## 一、研究主题概述

本报告围绕“计算机视觉在矿井安全监测中的大创选题”展开，目标是把软件工程专业学生可完成的系统开发能力，与矿井安全监测中的真实需求结合起来。推荐将题目收敛为：

**基于轻量化目标检测与低照度图像增强的矿井安全隐患视觉识别系统**

候选细分方向：

1. 矿井人员不安全行为识别：安全帽、反光服、跌倒、闯入危险区域、人与设备距离过近。
2. 输送带异物与异常状态检测：煤流异物、跑偏、堆煤、烟雾或火点早期线索。
3. 低照度矿井视频增强与检测协同：先增强再检测，或训练端加入低照度增强模块。
4. 矿井移动装备防碰撞视觉感知：行人、车辆、障碍物检测和距离预警。

软件工程专业的优势在于可以把算法、数据标注、模型训练、边缘部署、Web 可视化和报警联动做成完整 Demo，而不是只停留在单个模型指标上。

## 二、研判结论摘要

| 维度 | 结论 |
|---|---|
| 选题价值 | 高。矿井安全生产视频 AI 识别、井下人员/设备视觉感知、智能矿山边缘部署都是当前矿山智能化建设中的明确需求。 |
| 大创适配度 | 高。目标检测、图像增强、姿态估计、Web 看板都能形成可展示原型，适合本科项目周期。 |
| 创新空间 | 中高。不要泛泛做“矿井安全监测平台”，建议聚焦“低照度 + 小目标 + 粉尘遮挡 + 边缘部署”中的 1-2 个难点。 |
| 数据风险 | 中。真实井下数据难获取，应提前准备公开数据、仿真数据、现场替代数据和自采模拟数据方案。 |
| 推荐 baseline | Ultralytics YOLO 或 MMDetection。若强调工程落地，优先 YOLO；若强调算法对比和论文规范，补充 MMDetection。 |
| 导师匹配 | 建议采用“计算机/人工智能导师 + 安全/矿业场景导师”双导师思路。 |

最推荐的大创题目版本：

**基于 YOLO 与低照度图像增强的矿井人员安全行为实时识别系统**

该题目边界清晰，软件工程实现链条完整，演示性强，既能体现计算机视觉，又能落到矿井安全场景。

## 三、矿井应用知识图谱分析

MineIntel 知识图谱脚本检索命中“计算机视觉”“瓦斯监测监控”“顶板与矿压监测”“大型固定设备运维”“AI 大模型赋能”等节点。结合本选题，可提炼为以下矿井安全监测场景：

| 场景 | 视觉任务 | 价值 |
|---|---|---|
| 人员安全行为 | 安全帽/反光服/PPE 检测、跌倒检测、危险区域闯入识别 | 降低人工巡检依赖，提高违规行为发现速度 |
| 移动目标监测 | 人员、车辆、障碍物检测与跟踪 | 支撑井下防碰撞和区域安全预警 |
| 输送带安全 | 异物、堆煤、跑偏、烟雾火点视觉检测 | 对接煤流运输环节，Demo 容易做 |
| 顶板与巷道状态 | 裂隙、变形、支护异常视觉线索识别 | 可作为扩展方向，数据获取较难 |
| 固定设备运维 | 设备仪表、状态灯、异常烟雾或人员靠近检测 | 适合和边缘端巡检系统结合 |

矿井场景的特殊技术难点：

1. 低照度和强局部光源并存，普通检测模型容易漏检或误检。
2. 粉尘、水雾、遮挡、镜头污染会破坏图像纹理。
3. 人员和设备目标尺度变化大，小目标、远距离目标多。
4. 井下网络和算力受限，需要轻量化模型和边缘推理。
5. 安全场景容错率低，误报过多会影响使用，漏报又会带来安全风险。

对应技术方案：

1. 模型侧：YOLOv8/YOLOv10/YOLOv11、RT-DETR、轻量化 Backbone、注意力机制、小目标检测改进。
2. 图像侧：低照度增强、去雾/去粉尘、超分辨率、数据增强和合成扰动。
3. 系统侧：摄像头输入、边缘端推理、报警规则、Web 看板、日志追溯。
4. 评价侧：mAP、召回率、误报率、FPS、端侧延迟、低照度分组实验。

## 六、GitHub baseline 推荐

### 首选 baseline：Ultralytics YOLO

- 仓库：ultralytics/ultralytics
- 链接：https://github.com/ultralytics/ultralytics
- 语言：Python
- 适用原因：训练、验证、推理、导出 ONNX/TensorRT 的流程成熟，适合软件工程专业快速搭建端到端 Demo。
- 风险提示：正式参赛或商业化前需要核验许可证和模型版本使用限制。

迁移方案：

1. 用 YOLO 训练矿井安全目标类别：person、helmet、vest、fall、belt_foreign_object、smoke/fire 等。
2. 加入低照度增强预处理，对比“原图检测”和“增强后检测”。
3. 导出 ONNX，用边缘端或本机服务做实时推理。
4. 搭建 Web 看板，展示视频流、检测框、报警日志、置信度和处理延迟。

备选 baseline：

- open-mmlab/mmdetection：https://github.com/open-mmlab/mmdetection  
  适合做更规范的检测算法对比，但工程上手成本高于 YOLO。

## 八、技术路线建议

### 8.1 系统架构

1. 数据层：公开数据集、公开视频帧、自采模拟视频、合成低照度/粉尘增强数据。
2. 模型层：YOLO baseline + 低照度增强模块 + 小目标/遮挡数据增强。
3. 推理层：本机 GPU/CPU 推理，后期可导出 ONNX。
4. 应用层：Web 可视化看板、报警规则、截图留证、日志查询。
5. 评估层：mAP、Recall、FPS、误报率、低照度分组测试。

### 8.2 阶段计划

| 阶段 | 周期 | 任务 | 交付物 |
|---|---|---|---|
| 选题与调研 | 第 1-2 周 | 明确场景、查论文、确定数据和 baseline | 调研报告、开题 PPT |
| 数据准备 | 第 3-5 周 | 标注样本、构造低照度和粉尘增强数据 | 数据集、标注规范 |
| Baseline 复现 | 第 6-8 周 | 跑通 YOLO 训练、验证、视频推理 | 初版模型、检测 Demo |
| 改进实验 | 第 9-12 周 | 加入图像增强、小目标增强、轻量化对比 | 对比实验表 |
| 系统集成 | 第 13-16 周 | Web 看板、报警日志、模型导出 | 完整原型系统 |
| 结题答辩 | 第 17-18 周 | 报告、PPT、演示视频、软著/开源整理 | 结题材料 |

### 8.3 评价指标

1. 检测性能：mAP@0.5、mAP@0.5:0.95、Recall、Precision。
2. 安全性能：漏报率优先于误报率，关键类别需要单独统计。
3. 工程性能：FPS、端到端延迟、CPU/GPU 占用。
4. 鲁棒性：低照度、粉尘、遮挡、小目标分组指标。
5. 展示效果：实时视频推理、报警记录、截图留证、趋势统计。

## 九、科研经验参考

知乎/小红书检索脚本在当前环境仅返回公开搜索入口，不能作为论文事实依据。结合大创项目经验，可采用以下执行策略：

1. 题目要小而完整：优先做“人员安全行为识别”或“输送带异物检测”，不要一开始就做全矿井平台。
2. 先跑通 baseline：先用 YOLO 完成训练、验证和视频推理，再做模型改进。
3. 数据要尽早解决：数据来源是最大风险，至少准备公开数据、模拟采集、合成增强三套方案。
4. Demo 要稳定：答辩现场可运行的系统，比只展示一张模型结构图更有说服力。
5. 结果要可解释：把误检、漏检、低照度失败案例单独做成分析页，体现工程反思。
6. 团队分工要清晰：软件工程学生可负责后端推理服务、前端看板、数据标注工具和部署脚本。

## 十、申报书写法建议

### 10.1 项目名称

推荐：

**基于轻量化目标检测的矿井人员安全行为视觉识别与预警系统**

备选：

1. 基于低照度图像增强与 YOLO 的矿井安全隐患识别系统
2. 面向矿井复杂环境的人员安全行为视觉检测系统
3. 基于边缘视觉智能的矿井输送带异常检测与报警系统

### 10.2 创新点

1. 面向矿井低照度、粉尘、遮挡环境构造专用数据增强策略。
2. 将轻量化目标检测模型部署到实时视频预警系统中。
3. 建立“检测结果 + 报警日志 + 截图留证”的可追溯安全监测闭环。
4. 对关键安全类别按漏报风险进行单独评估。

### 10.3 风险与对策

| 风险 | 对策 |
|---|---|
| 真实井下数据难获取 | 使用公开数据 + 模拟采集 + 数据增强，必要时把研究目标收敛为方法验证 |
| 模型效果不稳定 | 先保证 baseline，后做低照度增强和小目标优化 |
| 端侧实时性不足 | 选择 YOLO nano/small 版本，导出 ONNX，降低输入分辨率 |
| 答辩演示出错 | 准备离线视频、录屏和静态案例三套展示方式 |

## 十三、核心结论

对软件工程专业学生而言，“计算机视觉在矿井安全监测中的大创选题”最稳妥的落点不是做泛化的智慧矿山平台，而是做一个可训练、可推理、可展示、可报警的细分视觉系统。推荐以 YOLO baseline 为起点，聚焦人员安全行为或输送带异常检测，再把低照度增强和边缘部署作为创新点。这样既有矿业场景价值，也符合软件工程专业能够交付完整系统的优势。
